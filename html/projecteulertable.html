Future Plans

The assembly language implementations deliberately, at the first pass, only use the integer unit of the ARM processor. Later I intend to investigate the use of the floating point unit in conjunction with the integer unit to improve performance. The integer-only implementation is useful for those ARM processors without a floating-point unit (e.g. the SheevaPlug). Optimisation needs a baseline to work from (we can see where we are deficient by comparison with the C compilers).

We can also compare the gcc compiler to that provided by LLVM (clang).
